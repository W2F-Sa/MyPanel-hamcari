# MyPanel — پنل نمایندگی برای 3x-ui

یک پنل **نمایندگی (Reseller)** امن که روی پنل **3x-ui (سنایی)** شما سوار می‌شود و از طریق
API رسمی آن کار می‌کند. دو پرتال جداگانه دارد:

- **پنل مدیر (Admin):** مدیریت پنل‌های 3x-ui، ساخت نماینده، تعیین قیمت اختصاصی هر گیگ،
  افزایش/کاهش اعتبار، مشاهده‌ی کاربران و گزارش‌ها.
- **پنل نماینده (Agent):** نماینده فقط با یک **توکن** وارد می‌شود، با انتخاب حجم (۱ تا سقف تعیین‌شده)
  و تعداد روز، کاربر می‌سازد و هزینه از اعتبارش کسر می‌شود؛ لینک‌های اتصال بلافاصله آماده‌اند.

> این برنامه **بدون nginx/apache** خودش HTTPS را سرو می‌کند و روی یک پورت دلخواه (پیش‌فرض `2053`)
> بالا می‌آید، تا با پورت‌های ۸۰/۴۴۳ که درگیر 3x-ui هستند تداخل نکند.

---

## نصب سریع (از گیت‌هاب، مستقیم)

روی سرور (به‌عنوان root):

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/W2F-Sa/MyPanel-hamcari/main/install.sh)
```

برای پورت دلخواه:

```bash
PORT=2053 bash <(curl -fsSL https://raw.githubusercontent.com/W2F-Sa/MyPanel-hamcari/main/install.sh)
```

نصب‌کننده به‌صورت خودکار:

1. Node.js (نسخه ≥ ۱۸) و پیش‌نیازها را نصب می‌کند.
2. کد را در `/opt/mypanel-hamcari` می‌گیرد و وابستگی‌ها را نصب می‌کند.
3. یک گواهی **TLS خودامضا** می‌سازد (پنل خودش HTTPS می‌دهد).
4. **نام کاربری/رمز تصادفی مدیر** و **دو مسیر مخفی** برای پرتال‌ها می‌سازد و چاپ می‌کند.
5. **پلن‌های پیش‌فرض** (۳۰/۶۰/۹۰/۱۸۰/۳۶۵ روزه) و **پنل اصلی 3x-ui شما** را به‌صورت پیش‌فرض اضافه می‌کند.
6. سرویس `systemd` را با یک کاربر بدون‌دسترسی نصب و اجرا می‌کند.
7. پورت را در فایروال (ufw/firewalld) باز می‌کند.

در پایان، آدرس هر دو پرتال و اطلاعات ورود مدیر چاپ می‌شود — آن‌ها را ذخیره کنید.

> **پنل پیش‌فرض:** هنگام اولین اجرا، پنل 3x-ui شما (همان آدرس و توکنی که دادید) خودکار اضافه می‌شود تا
> بلافاصله آماده‌ی کار باشد. می‌توانید با متغیرهای محیطی آن را تغییر دهید یا غیرفعال کنید:
> `MYPANEL_SEED_PANEL_URL`، `MYPANEL_SEED_PANEL_TOKEN` (مقدار خالی = بدون پنل پیش‌فرض)، `MYPANEL_SEED_PANEL_NAME`.
>
> **هشدار امنیتی:** توکن پنل پیش‌فرض داخل کد قرار دارد. توصیه می‌شود پس از نصب، در 3x-ui یک توکن تازه بسازید
> و از پنل مدیر (ویرایش پنل) جایگزین کنید، سپس توکن قدیمی را باطل کنید.

> اطلاعات ورود مدیر فقط یک‌بار هنگام نصب چاپ می‌شود. اگر گمش کردید:
> `cd /opt/mypanel-hamcari && node src/cli/resetAdmin.js`

## راه‌اندازی گام‌به‌گام

### ۱) افزودن پنل 3x-ui
وارد **پنل مدیر** شوید → بخش «پنل‌ها» → «افزودن پنل»:

- **آدرس کامل پنل:** شامل پروتکل، دامنه/آی‌پی، پورت و مسیر (base path). مثال:
  `https://your-host:2087/AbCdEf123`
- **توکن API:** از مسیر *Settings → Security → API Token* در خود 3x-ui بسازید و اینجا بگذارید.
- اگر گواهی پنل خودامضاست، گزینه‌ی **«اجازه‌ی گواهی self-signed»** را روشن کنید.

با دکمه‌ی **«تست اتصال»** صحت آدرس/توکن را بسنجید (وضعیت Xray و نسخه را نشان می‌دهد).

### ۲) ساخت نماینده
بخش «نمایندگان» → «نماینده جدید»:

- **پنل** را انتخاب کنید؛ سپس **اینباندهای مجاز** (مثلاً اینباند ۱، ۲ و ۳) را برای فروش تیک بزنید.
- **قیمت هر گیگ** (تومان، فقط عدد صحیح)، **سقف گیگ هر کاربر**، **روز پیش‌فرض** و **اعتبار اولیه** را تعیین کنید.
- پس از ساخت، یک **توکن ورود** نمایش داده می‌شود که **فقط همان یک‌بار** دیده می‌شود؛ آن را برای نماینده بفرستید.

### ۳) تعریف پلن (مدت‌زمان)
بخش «پلن‌ها» در پنل مدیر:

- هر پلن یک **نام** و یک **مدت‌زمان به روز** دارد (مثلاً «یک‌ماهه = ۳۰ روز»).
- **مدت‌زمان فقط دست شماست**؛ نماینده هنگام ساخت کاربر صرفاً یکی از پلن‌ها را انتخاب می‌کند و **حجم** را خودش تعیین می‌کند.
- هزینه = حجم × قیمت هر گیگِ نماینده (پلن روی هزینه اثر ندارد، فقط مدت را تعیین می‌کند).
- می‌توانید پلن‌ها را فعال/غیرفعال یا حذف کنید. در نصب اولیه چند پلن پیش‌فرض ساخته می‌شود.

### ۴) شارژ نماینده
در ردیف هر نماینده دکمه‌ی **«شارژ»**: می‌توانید اعتبار را **افزایش** یا **کاهش** دهید (عدد صحیح، تومان).
هر تغییر در تاریخچه‌ی تراکنش‌ها ثبت می‌شود.

### ۵) کار با پنل نماینده
نماینده وارد **پرتال نمایندگی** می‌شود (فقط با توکن) و:

- در «ساخت کاربر» یک **پلن** (مدت) را انتخاب و **حجم** (۱ تا سقف) را تعیین می‌کند؛ هزینه به‌صورت زنده محاسبه و از اعتبارش کسر می‌شود.
- کاربر روی **همه‌ی اینباندهای مجاز** ساخته می‌شود و برای هر اینباند یک **لینک اتصال** تولید می‌شود.
- می‌تواند لینک‌ها را کپی کند، کاربر را **شارژ مجدد** (افزودن حجم و/یا تمدید با یک پلن) یا حذف کند.

## افزودن اینباند جدید بدون باگ

اینباندهای مجاز هر نماینده در پروفایلش ذخیره می‌شوند. هر زمان روی پنل 3x-ui اینباند جدیدی ساختید
(مثلاً اینباند ۴)، کافی است در **پنل مدیر → ویرایش نماینده** آن اینباند را هم تیک بزنید.
کاربرانی که **از آن پس** ساخته می‌شوند روی همه‌ی اینباندهای انتخاب‌شده قرار می‌گیرند.
شناسه‌ها همیشه به‌صورت **عدد صحیح** خوانده می‌شوند و تعداد اینباند محدودیتی ندارد.

> تمام مقادیر عددی (حجم، قیمت، اعتبار، روز) در سراسر برنامه **فقط عدد صحیح**اند؛ ورودی اعشاری رد می‌شود.

## امنیت

- **دو مسیر مخفی و تصادفی** برای پرتال‌ها؛ مسیرهای ناشناخته `404` می‌گیرند و چیزی لو نمی‌رود.
- نشست‌ها سمت سرور و کوکی **HttpOnly + Secure + SameSite=Strict**؛ شناسه‌ی نشست به‌صورت هش ذخیره می‌شود.
- محافظت **CSRF** برای همه‌ی درخواست‌های تغییردهنده.
- **محدودسازی نرخ ورود** (قفل موقت پس از چند تلاش ناموفق).
- رمز مدیر با **bcrypt** هش می‌شود؛ توکن‌های API پنل‌ها با **AES-256-GCM** رمزنگاری و ذخیره می‌شوند.
- توکن نماینده فقط هش آن نگه‌داری می‌شود (هش SHA-256).
- هدرهای امنیتی سخت‌گیرانه (CSP، HSTS، X-Frame-Options=DENY و …).
- سرویس با یک **کاربر بدون‌دسترسی** و سخت‌سازی systemd اجرا می‌شود.

> **یادآوری مهم:** هر توکن API که جایی به‌اشتراک گذاشته‌اید را در 3x-ui باطل (Delete/Disable) و توکن تازه بسازید.

## رابط کاربری

طراحی **شیشه‌ای (glassmorphism)**، مدرن، **راست‌چین (فارسی)**، کاملاً **ریسپانسیو** و دارای
**حالت روز/شب**. پالت رنگ فیروزه‌ای/زمردی (بدون بنفش).

## پیکربندی

تنظیمات در `data/config.json` ذخیره می‌شود (هنگام init ساخته می‌شود):

| کلید | توضیح | پیش‌فرض |
|------|-------|---------|
| `port` | پورت شنونده | `2053` |
| `adminPath` / `resellerPath` | مسیرهای مخفی پرتال‌ها | تصادفی |
| `tls.enabled` | سرو مستقیم HTTPS | `true` |
| `tls.certFile` / `tls.keyFile` | مسیر گواهی/کلید | `data/certs/*` |
| `sessionTtlHours` | عمر نشست | `12` |
| `loginMaxAttempts` | سقف تلاش ورود قبل از قفل | `6` |
| `trustProxy` | اعتماد به هدرهای `X-Forwarded-*` (فقط پشت پروکسی مطمئن) | `false` |

متغیرهای محیطی مثل `MYPANEL_PORT`, `MYPANEL_ADMIN_PATH`, `MYPANEL_RESELLER_PATH` مقادیر فایل را override می‌کنند.

### اجرای پشت پروکسی (اختیاری)
اگر می‌خواهید پشت یک TLS terminator اجرا کنید، با `MYPANEL_HTTP=1` برنامه HTTP ساده می‌دهد و
`trustProxy` را در config روشن کنید.

## عیب‌یابی

- **سرویس بالا نمی‌آید:** `journalctl -u mypanel -f`
- **خطای TLS در مرورگر:** گواهی خودامضاست؛ یک‌بار استثنا را بپذیرید یا گواهی معتبر در `data/certs` بگذارید.
- **تست پنل ناموفق:** آدرس کامل (با پورت و مسیر) و توکن را بررسی کنید؛ برای گواهی خودامضای پنل، گزینه‌ی self-signed را روشن کنید.
- **«اعتبار کافی نیست»:** اعتبار نماینده را از پنل مدیر افزایش دهید.

## ساختار پروژه

```
src/
  server.js            نقطه‌ی ورود (HTTPS، روتینگ پرتال‌ها، میدلورها)
  config.js            بارگذاری/ذخیره‌ی پیکربندی
  cli/                 اسکریپت‌های init و reset-admin
  lib/                 db, crypto, http, sessions, security, validate, fetch, audit
  middleware/auth.js   احراز هویت admin/reseller
  routes/              مسیرهای API دو پرتال
  services/            منطق پنل‌ها، نمایندگان، کاربران، کلاینت 3x-ui
public/assets/         رابط کاربری (CSS و JS، بدون وابستگی خارجی)
install.sh             نصب‌کننده‌ی خودکار
```
